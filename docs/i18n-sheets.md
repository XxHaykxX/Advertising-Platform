# i18n ↔ Google Sheets: цикл работы контент-райтера

Все тексты интерфейса сайта (775 ключей × hy/ru/en) живут в `src/lib/i18n.ts`.
Райтер правит их в Google-таблице; скрипты гоняют данные туда-обратно.
Google API и ключи не нужны — импорт идёт по публичной CSV-ссылке листа.

## Разовая настройка

1. Создать Google-таблицу. Доступ: «Все, у кого есть ссылка» → **Читатель**;
   райтера добавить **Редактором** по email.
2. `npm run i18n:export` → появляется `i18n-export.csv` (гитигнорен).
3. В таблице: Файл → Импорт → Загрузка → выбрать CSV → «Заменить текущий лист».
4. В `.env` добавить CSV-ссылку листа (НЕ обычную ссылку на таблицу):

   ```
   I18N_SHEET_CSV_URL=https://docs.google.com/spreadsheets/d/<ID>/export?format=csv
   ```

   `<ID>` — длинный код из адреса таблицы. Если нужный лист не первый,
   добавить `&gid=<gid листа>`.

## Рабочий цикл

1. Райтер правит колонки `hy` / `ru` / `en` прямо в таблице.
2. `npm run i18n:import` — скрипт скачивает таблицу, валидирует и точечно
   переписывает изменённые ключи в `src/lib/i18n.ts` (дифф минимальный,
   секционные комментарии сохраняются).
3. `npx vitest run` (guard-тест словаря) + `npx tsc --noEmit`.
4. Commit + push в `main` → автодеплой Hostinger.

Импорт **отказывается** трогать файл и печатает список проблем, если в
таблице: удалена/добавлена строка, изменён `key`, пустая ячейка, потерян
`{плейсхолдер}`, кириллическая буква внутри армянского текста (гомоглиф).

## Памятка райтеру (вставить в шапку таблицы или отправить письмом)

- Править можно **только** колонки `hy`, `ru`, `en`.
- Колонку `key` не менять, строки не удалять и не добавлять.
- Слова в фигурных скобках — `{n}`, `{name}` — это подстановки: не переводить,
  не удалять, оставлять как есть в любом языке.
- Пустых ячеек не оставлять.
- В армянский текст не вставлять русские буквы (частая ошибка при копипасте —
  русская «о»/«а» вместо армянской буквы; импорт такое отловит и укажет ключ).

## Когда разработчики добавили новые ключи

Порядок строгий, иначе потеряются правки райтера:

1. `npm run i18n:import` — сначала забрать всё, что райтер уже наделал.
2. `npm run i18n:export` — свежий CSV уже с новыми ключами.
3. Переимпортировать CSV в таблицу («Заменить текущий лист»).

## Что НЕ входит в таблицу

- Контент из БД: названия и синопсисы проектов, портфолио — редактируются
  в админке, там уже есть поля для всех трёх языков.
- Одноязычные поля БД (студия, tagline, названия партнёров) — пока без i18n.
- Тексты админ-панели — она намеренно англоязычная.

## Файлы

- `scripts/i18n-export.ts` — словарь → `i18n-export.csv`
- `scripts/i18n-import.ts` — Google Sheet → `src/lib/i18n.ts`
- `scripts/i18n-lib.ts` — общий CSV-парсер/сериализатор, карта контекстов, проверки
- Финальная защита: `src/lib/i18n.guard.test.ts`
