# Кинематографичный 3D-апгрейд сайта AD PLACEMENT — мануал + промпт

Документ из двух частей:
- **Часть 1 — Мануал/план**: как превратить наш текущий сайт (Next.js + Tailwind + framer-motion + Lenis) в полноценный сайт с 3D-эффектами по всем секциям (как страница `/experience`, но везде).
- **Часть 2 — Готовый master-prompt**: большой промпт, который можно вставить в AI-агента (Claude Code / Cursor / v0), чтобы он сделал этот апгрейд.

## ЧАСТЬ 1 — МАНУАЛ / ПЛАН

### 1.1. Идея

Сейчас сайт 2D: секции анимируются через framer-motion (parallax, reveal, marquee). Цель — **3D-слой на весь сайт**: единая Three.js-сцена под контентом, по которой **камера летит, управляемая скроллом**. Каждая секция = «остановка» камеры в 3D-пространстве. DOM-контент (тексты, кнопки, формы) остаётся сверху как оверлей.

Главный принцип: **скролл = движение камеры**. Один `<canvas>` на весь сайт (не по одному на секцию — это убьёт производительность).

### 1.2. Стек

| Слой | Библиотека | Зачем |
|------|-----------|-------|
| Фреймворк | Next.js 16 (App Router) | уже есть |
| Стили | Tailwind v4 + shadcn/ui | уже есть |
| 2D-анимации | framer-motion | reveal, оверлеи, UI |
| Плавный скролл | **Lenis** | основа: даёт `progress` 0..1 |
| 3D-рендер | **three** + **@react-three/fiber** | сцена, камера, меши |
| 3D-хелперы | **@react-three/drei** | текстуры, Text, Image, useTexture |
| Пост-обработка | **@react-three/postprocessing** | Bloom (свечение), Vignette, DOF |
| Доп. триггеры (опц.) | **GSAP + ScrollTrigger** | точные scroll-сцены, пиннинг |

Установка нового:
```bash
npm i three @react-three/fiber @react-three/drei @react-three/postprocessing
npm i -D @types/three
```

### 1.3. Архитектура

```
RootLayout
 └─ <SmoothScroll> (Lenis root)            ← даёт глобальный progress
     ├─ <Scene3DProvider>                  ← один <Canvas> на весь сайт
     │    └─ fixed inset-0 -z-10 canvas    ← рендерит ВСЕ 3D-сцены
     ├─ <SiteHeader/>
     ├─ {page DOM-контент: секции}         ← прозрачный фон, лежит ПОВЕРХ canvas
     └─ <SiteFooter/>
```

Ключ: **canvas — фиксированный фон (`position:fixed; inset:0; z-index:-10`)**, а DOM-секции прокручиваются поверх него. Каждая секция знает свой диапазон скролла; 3D-сцена реагирует на общий `progress`.

### 1.4. Ключевой паттерн скролла (важно!)

Мы это уже выяснили на `/experience`: **нельзя использовать drei `ScrollControls` вместе с глобальным Lenis** — Lenis перехватывает колесо, ScrollControls остаётся без событий. Правильно:

```tsx
// progress пишется из Lenis, читается в useFrame без ре-рендеров
const progress = useRef(0);
useLenis((lenis) => { progress.current = lenis.progress || 0; });

function CameraRig({ progress }) {
  useFrame((state) => {
    const off = progress.current;                  // 0..1 по всей странице
    const cam = state.camera;
    cam.position.z += (targetZ(off) - cam.position.z) * 0.08; // плавный lerp
    cam.position.x += (state.pointer.x * 1.1 - cam.position.x) * 0.04; // parallax мыши
    cam.lookAt(0, 0, cam.position.z - 6);
  });
  return null;
}
```

Для пер-секционных эффектов: каждая секция получает свой локальный progress через `IntersectionObserver` или `framer-motion useScroll({ target: ref })`, и передаёт его в свою 3D-группу.

### 1.5. Карта секций → 3D-эффект

| Секция | 3D-приём |
|--------|----------|
| **Hero** | Камера летит сквозь спиральный коридор постеров (готово на `/experience`) — сделать стартом. Bloom + туман. |
| **Как это работает** | 3 «станции» в 3D-пространстве. Камера переезжает от станции к станции по скроллу; timeline-линия — настоящая 3D-линия/трубка, рисуется. Иконки шагов — 3D-объекты (хлопушка, планшет, гарнитура) или подсвеченные плоскости. |
| **Каталог** | 3D-полка/стена постеров с настоящей глубиной. Hover = карточка наклоняется в 3D (tilt), выезжает к камере. Можно «карусель-кольцо» проектов. |
| **Портфолио** | 3D-галерея: кейсы расставлены в пространстве, при клике — 3D-доли/переход (shared transition) в лайтбокс. |
| **Партнёры** | 3D-кольцо логотипов, вращается; либо бесконечная 3D-лента в перспективе. |
| **Контакты** | Лёгкий 3D-фон: парящие частицы/пыль, depth-of-field, форма — обычный DOM поверх. |
| **Переходы между секциями** | Непрерывный полёт камеры: одна сцена, разные «зоны» по оси Z, скролл провозит сквозь все. Контент-оверлеи появляются в нужных диапазонах. |

### 1.6. Ассеты

- **Сейчас (бюджетно):** постеры-картинки как текстуры на плоскостях (как у нас в `/experience`). Работает, выглядит кинематографично.
- **«Настоящий» рендер (как Lapz):** нужен один из:
  - **Image-sequence** — пре-рендер 150–300 кадров из Blender/C4D, скраб по скроллу на `<canvas>`. Тяжёлые ассеты, зато фотореализм.
  - **GLB/GLTF-модели** сцен/объектов (Draco-сжатие, KTX2-текстуры), камера летит по сцене. Гибко, нужен 3D-художник.
- Картинки оптимизировать: WebP/AVIF, KTX2 для текстур, `draco` для геометрии.

### 1.7. Производительность

- **Один Canvas** на весь сайт, не несколько.
- `frameloop="demand"` где возможно; иначе ограничить `dpr={[1, 1.5]}`.
- **Instancing** для множества одинаковых объектов (частицы, повторяющиеся рамки).
- Lazy-загрузка тяжёлых сцен (`next/dynamic`, `Suspense`).
- Текстуры: KTX2 + `anisotropy` умеренно.
- Скрывать (`visible={false}`), а не размонтировать сцены вне зоны.
- Бюджет кадра < 16 мс (60 fps). Мониторить drei `<PerformanceMonitor>` + `AdaptiveDpr`.

### 1.8. Доступность и фолбэки (обязательно)

- **`prefers-reduced-motion`** → отключить полёт камеры, показать статичную сцену/2D-версию.
- **Нет WebGL / слабое устройство** → детект, рендерить текущий 2D-сайт (он у нас уже есть — это идеальный фолбэк!).
- **Мобайл** → упрощённая сцена (меньше объектов, ниже dpr), либо 2D.
- Контент (тексты/формы/ссылки) — всегда в DOM, доступен скринридерам, не внутри canvas.
- SEO: текст в DOM, не в 3D (Text-в-3D не индексируется).

### 1.9. Фазовый план работ

1. **Фундамент:** глобальный `<Scene3DProvider>` (один fixed canvas) + проводка `lenis.progress`. Фолбэк-детект WebGL/reduced-motion.
2. **Hero → 3D:** перенести коридор постеров из `/experience` в главный Hero как первую зону камеры.
3. **Зоны камеры:** разметить ось Z по секциям, синхронизировать появление DOM-оверлеев с диапазонами скролла.
4. **По секции за раз:** Как-это-работает → Каталог → Портфолио → Партнёры → Контакты, добавляя 3D-приём из таблицы 1.5.
5. **Пост-обработка:** Bloom + Vignette + DOF (аккуратно, чтобы не убить fps).
6. **Оптимизация + фолбэки:** instancing, KTX2/Draco, reduced-motion, mobile, WebGL-детект.
7. **QA:** Playwright-скриншоты (как `scripts/verify-experience.mjs`), проверка fps, мобайл, reduced-motion.

### 1.10. Чеклист готовности

- [ ] Один Canvas на весь сайт, fixed-фон.
- [ ] Скролл-камера на `lenis.progress` (без ScrollControls под Lenis).
- [ ] Каждая секция имеет свою 3D-зону + DOM-оверлей.
- [ ] Bloom/туман/виньетка — кинематографично, 60 fps.
- [ ] `prefers-reduced-motion` → 2D-фолбэк.
- [ ] Нет WebGL → текущий 2D-сайт.
- [ ] Мобайл упрощён.
- [ ] Контент в DOM (SEO/a11y).
- [ ] Тёмная Netflix-палитра, красный акцент `#E50914`.

## ЧАСТЬ 3 — ВЫБРАННЫЙ ПОДХОД: ПОКАДРОВЫЙ СКРАБ + Nano Banana Pro / Kling

> Исходя из исследования (Apple AirPods и подобные = не живой 3D, а **скраб кадров**) и доступных
> у заказчика инструментов: **Nano Banana Pro** (генерация кадров по промту) + **Kling**
> (видео из первого и последнего кадра). Это наш основной маршрут. Живой R3F (`/experience`)
> остаётся как доп-страница.

### 3.1. Почему этот подход
- Фотореализм + кинематографичность из коробки (кадры рисует AI).
- **Не нужен WebGL** → плавно везде, а наш текущий 2D-сайт = идеальный фолбэк.
- Совпадает с инструментами заказчика (Nano Banana Pro + Kling).
- Минус: ассеты тяжёлые, камера по фикс-пути (не интерактивно) — для лендинга ок.

### 3.2. Пайплайн ассетов (делает заказчик в AI)
1. **Nano Banana Pro** — сгенерить опорные кадры сцены по промту (минимум первый и последний;
   лучше 2–4 опорных). Один стиль/камера/свет на всю сцену.
2. **Kling** — режим «первый + последний кадр»: на вход два кадра → на выходе плавное
   видео-переход (4–10 сек). По одному клипу на сцену/секцию.
3. **ffmpeg** — видео → последовательность кадров:
   ```bash
   ffmpeg -i scene.mp4 -vf "fps=30,scale=1920:-1" public/sequences/hero/%04d.webp
   ```
   (≈120–180 кадров на сцену, одинаковый размер.)
4. **Оптимизация** — WebP/AVIF, ширина 1600–1920, сжать. Цель: сцена в несколько МБ.
5. Складировать: `public/sequences/<scene>/0001.webp …`.

### 3.3. Структура сцен
- Вариант рекоменд.: **по последовательности на секцию** (Hero, фон Каталога, фон Портфолио…),
  100–180 кадров каждая, ленивая загрузка по приближении.
- Альтернатива: одна длинная мастер-последовательность на весь сайт + оверлеи по диапазонам.

### 3.4. Код: скраб кадров на `<canvas>` от Lenis

```tsx
"use client";
import { useEffect, useRef } from "react";
import { useLenis } from "lenis/react";

// frames: ["/sequences/hero/0001.webp", ...]  (предгенерированы)
export function SequenceScrub({ frames }: { frames: string[] }) {
  const canvasRef = useRef<HTMLCanvasElement>(null);
  const images = useRef<HTMLImageElement[]>([]);
  const frame = useRef(0);
  const progress = useRef(0);

  // preload
  useEffect(() => {
    images.current = frames.map((src) => { const i = new Image(); i.src = src; return i; });
  }, [frames]);

  // прогресс из Lenis (0..1 по странице или по секции)
  useLenis((l) => { progress.current = l.progress || 0; });

  // рисуем нужный кадр (cover) только при смене
  useEffect(() => {
    const cv = canvasRef.current!; const ctx = cv.getContext("2d")!;
    let raf = 0, last = -1;
    const draw = () => {
      const N = frames.length;
      const idx = Math.min(N - 1, Math.max(0, Math.round(progress.current * (N - 1))));
      if (idx !== last && images.current[idx]?.complete) {
        const img = images.current[idx];
        const cw = cv.width = cv.clientWidth, ch = cv.height = cv.clientHeight;
        const r = Math.max(cw / img.width, ch / img.height);
        const w = img.width * r, h = img.height * r;
        ctx.drawImage(img, (cw - w) / 2, (ch - h) / 2, w, h);
        last = idx;
      }
      raf = requestAnimationFrame(draw);
    };
    raf = requestAnimationFrame(draw);
    return () => cancelAnimationFrame(raf);
  }, [frames]);

  return <canvas ref={canvasRef} className="fixed inset-0 -z-10 h-full w-full" />;
}
```
- Источник прогресса — наш **Lenis** (как на `/experience`).
- Для скорости: `createImageBitmap` / `OffscreenCanvas`, рисовать только при смене кадра.
- Прелоадер: показывать процент загрузки кадров перед стартом.

### 3.5. Готовые промпты для Nano Banana Pro (под наш бренд)

**Базовый стиль (добавлять в каждый промт):**
```
dark cinematic film still, near-black #0B0B0B background, deep shadows, moody volumetric
lighting, subtle red #E50914 accent glow, anamorphic lens, film grain, premium, high detail,
no text, no watermark, 16:9
```

**Hero — первый кадр:**
```
[базовый стиль] wide establishing shot flying low through a dark cinematic corridor lined with
glowing movie poster frames receding into red-lit fog, camera far back at the entrance
```
**Hero — последний кадр:**
```
[базовый стиль] same corridor of glowing movie poster frames, camera now deep inside surrounded
by the frames, one large poster filling the right side, red glow stronger
```
(Первый и последний кадр — одна сцена/свет/камера, меняется только положение камеры → Kling
плавно интерполирует.)

### 3.6. Kling — настройка
- Режим **first frame + last frame**.
- Длительность 5 c, плавное движение камеры, без резких склеек.
- Важно: оба кадра из одной сцены (один свет/палитра/композиция), иначе интерполяция «поедет».
- Сгенерить → скачать mp4 → в ffmpeg (3.2).

### 3.7. Фолбэки
- `prefers-reduced-motion` или медленная сеть → показать первый кадр статикой (или текущую 2D-секцию).
- Нет `<canvas>`/ошибка загрузки → 2D-сайт.
- Мобайл: меньше кадров (60–90) и/или меньшая ширина.

### 3.8. Фазы внедрения
1. **Тест-сцена (Hero):** Nano Banana 2 кадра → Kling → ffmpeg → ~120 WebP в `public/sequences/hero/`.
