# Release-notes формат (igovazd.am)

Триггер: пользователь говорит **«Дай мне информацию нового релиза»** (или похоже).
Действие: выдать changelog **на чистом армянском (hy)**, в шаблоне ниже. Коротко, ясно,
по-человечески, без технического жаргона. Сам этот документ — на русском; **армянский только в самом ответе-релизе.**

## Шаблон вывода

```
Թողարկում v{X.Y} — {YYYY} թ. {ամիս-hy}ի {D}

Ի՞նչ նորություն կա, և ի՞նչ շտկվեց igovazd.am-ում.

1. {Կարճ վերնագիր}։ {1–2 простых предложения — что нового / что починили.}
2. ...
```

Правила:
- Первая строка: `Թողարկում v{версия} — {дата на армянском}` (напр. `2026 թ. հուլիսի 17`).
- Вторая строка всегда: `Ի՞нч նорություն կա, և ի՞нч штквец igovazd.am-ум.`
- Нумерованный список; каждый пункт — жирный короткий заголовок + 1–2 простых предложения.
- Только пользовательские изменения (фичи + фиксы). БЕЗ тех-деталей (prisma, tsc, schema, postcss-версии и т.п. — писать «обновление безопасности», не имена пакетов).
- Чистый армянский шрифт, БЕЗ кириллических гомоглифов.
- Версию инкрементить от предыдущей.
