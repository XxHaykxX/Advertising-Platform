# Техническое задание (ТЗ)
## Advertising Platform — платформа для продажи product placement в фильмах

> **Версия:** 1.0  
> **Дата:** 10.06.2026  
> **Рабочее название:** Advertising Platform *(финальное название и домен — на уточнении)*  
> **Тип проекта:** Landing page + Admin panel (MVP)  
> **Исполнитель:** заказчик самостоятельно с применением AI-инструментов (21st.dev Magic и др.)

---

## 1. Общая информация

### 1.1. Назначение
Платформа — **посредник (marketplace / lead-gen)** между производителями фильмов и проектов (продавцы рекламных мест) и рекламодателями (покупатели product placement). Сайт выполняет роль **витрины проектов** и **канала лидогенерации**: покупатель находит подходящий проект и оставляет заявку либо звонит, после чего **менеджер связывается и ведёт сделку вручную** (по телефону / в мессенджере).

### 1.2. Ключевая бизнес-логика
- Производитель планирует **дату выхода** фильма/проекта (может быть в будущем — например, через 5 месяцев) и заранее передаёт информацию платформе.
- Администратор **заранее публикует** проект и доступные рекламные возможности (placement-слоты) на сайте через админ-панель.
- Покупатель видит **будущие проекты** и может **забронировать placement заранее**, до начала съёмок / выхода.
- По каждому проекту указывается **дедлайн приёма заявок** (после него бронирование закрыто) — это создаёт срочность.
- Монетизация в MVP не автоматизирована: оплаты и комиссии вне сайта, сделку закрывает менеджер.

### 1.3. Цели MVP
1. Презентовать платформу и доступные проекты для размещения рекламы.
2. Собирать заявки от покупателей (lead generation).
3. Дать администратору полный контроль над контентом без участия разработчика.

## 2. Роли пользователей

| Роль | Описание | Доступ |
|------|----------|--------|
| **Гость / Покупатель** (рекламодатель) | Просматривает каталог проектов, портфолио, партнёров; фильтрует проекты; оставляет заявку; связывается по телефону/мессенджеру | Публичная часть сайта |
| **Администратор / Менеджер** | Управляет всем контентом сайта, обрабатывает заявки | Admin panel (по паролю) |
| **Производитель фильма** | На старте кабинета не имеет — данные о проектах вносит администратор. *Личный кабинет производителя — Этап 2* | — (offline, через менеджера) |

## 3. Языки и локализация (i18n)

- **Три языка:** Русский (RU), Английский (EN), Армянский (HY).
- Переключатель языков в шапке и футере; выбор сохраняется (cookie / localStorage).
- Переводится **весь контент**: интерфейс, тексты лендинга, названия и описания проектов, портфолио.
- Корректная поддержка **армянского шрифта** (например, Noto Sans Armenian / системные).
- Язык по умолчанию — Русский *(можно изменить; на уточнении)*.
- Для SEO желательны `hreflang`-теги (опционально).

## 4. Карта сайта (Sitemap)

**Главная страница (one-page scroll):**
1. Header (шапка с навигацией и переключателем языка)
2. Hero (главный экран)
3. «Как это работает» (3 шага)
4. Каталог проектов (с фильтрами и поиском)
5. Портфолио (кейсы размещений)
6. Партнёры (слайдер логотипов)
7. Контакты + форма заявки
8. Footer (подвал)

**Дополнительно:**
- Страница / модальное окно проекта (детали фильма)
- Юридические страницы: **Политика конфиденциальности**, **Согласие на обработку персональных данных**

## 5. Функциональные требования — Лендинг (публичная часть)

### 5.1. Header (шапка)
- Логотип.
- Навигация по якорям (Как это работает, Каталог, Портфолио, Партнёры, Контакты).
- **Переключатель языка** RU / EN / HY.
- Кнопка **«Оставить заявку»** (CTA).
- Контакты быстрого доступа: **телефон** (кликабельный), иконки **WhatsApp / Telegram**.
- Sticky (закреплена при скролле), адаптивное бургер-меню на мобильных.

### 5.2. Hero (главный экран) — стиль Netflix
- **Кинематографичный, тёмная тема.**
- **Анимация и параллакс-эффект:** при скролле кадры/слои меняются (сменяющиеся кинокадры на фоне).
- Крупный заголовок + подзаголовок (УТП платформы) + кнопка CTA «Оставить заявку» / «Смотреть проекты».
- Адаптивность: на мобильных — облегчённая анимация для производительности.

### 5.3. «Как это работает» (3 шага)
1. **Производитель** публикует проект и доступные рекламные слоты заранее.
2. **Покупатель** выбирает проект и оставляет заявку.
3. **Менеджер платформы** связывается и сопровождает сделку.

### 5.4. Каталог проектов (фильмов/слотов)
Сетка карточек проектов с возможностью фильтрации.

**Карточка проекта содержит:**
- Постер / изображение.
- Название.
- Жанр.
- Тип размещения (placement): в кадре / упоминание / реквизит / сюжетная интеграция и т.п.
- Цена / бюджет — **опционально** (можно скрыть → «Цена по запросу»; при необходимости — мультивалютность).
- Статус слотов (всего / доступно).
- **Дата выхода** проекта.
- **Площадки показа:** YouTube, Kinodaran, TV и др.
- **Дедлайн приёма заявок** (до какой даты можно забронировать).

**Фильтры и поиск:**
- **Поиск по названию.**
- **Фильтр по жанру.**
- **Фильтр по сроку выхода** — на своё усмотрение предлагается: быстрые кнопки «в течение 1 / 3 / 6 месяцев / года» + выбор произвольного диапазона дат (от–до).
- Клик по карточке → **страница / модальное окно проекта**.

### 5.5. Страница проекта (детали)
- Все поля карточки + расширенное описание.
- Галерея (постер, кадры).
- Площадки, дата выхода, дедлайн бронирования, статус слотов.
- Кнопка **«Оставить заявку»** с автоподстановкой выбранного проекта в форму.

### 5.6. Портфолио (кейсы размещений)
- **Карточки-кейсы:** название бренда/фильма, описание (3 языка), медиа размещения.
- **Галерея фото + видео.**
- Видео двумя способами: **встраивание с YouTube** (по ссылке) **или загрузка файлом** на сервер.
- Цель — показать, где и в каких кадрах уже была размещена реклама.

### 5.7. Партнёры
- **Автопрокручивающийся слайдер (marquee)** с логотипами компаний-партнёров.
- Бесконечная плавная прокрутка, пауза при наведении (опционально), кликабельные логотипы (ссылка на партнёра).

### 5.8. Контакты + форма заявки
**Поля формы:**
- Имя — **обязательно**.
- Телефон — **обязательно** (главный канал: менеджер перезванивает).
- Email — опционально.
- Компания / бренд — опционально.
- Интересующий проект — select из каталога (автоподстановка при клике «Оставить заявку» на карточке).
- Бюджет — опционально.
- Сообщение / комментарий — опционально.
- **Чекбокс согласия** на обработку персональных данных (со ссылкой на юр. страницу) — обязательно.

**Способы связи (отображаются на сайте):** телефон, WhatsApp, Telegram.

**После отправки:** заявка сохраняется в админку **и** отправляется **уведомление на email**; пользователю — сообщение об успехе.

### 5.9. Footer (подвал)
- Логотип, краткая навигация, контакты, соцсети.
- Ссылки на юридические страницы.
- Переключатель языка, копирайт.

### 5.10. Юридические страницы
- **Политика конфиденциальности.**
- **Согласие на обработку персональных данных** (требуется, т.к. собираются ПД через форму).

## 6. Функциональные требования — Admin panel

Защищённая паролем панель управления, позволяющая менять весь контент сайта без разработчика.

### 6.1. Авторизация
- Вход по паролю, серверная сессия (cookie, httpOnly).
- Защита от перебора (rate limiting).
- **Смена пароля** администратора из панели.

### 6.2. Тексты и заголовки
- Редактирование **всех текстов лендинга** (Hero, шаги, заголовки секций, футер и т.д.).
- Каждое поле — **на трёх языках** (RU / EN / HY).

### 6.3. Проекты / фильмы (CRUD)
Создание, редактирование, удаление, сортировка, включение/выключение. Поля:
- Название, жанр, описание, тип размещения — **на 3 языках**.
- Постер / изображение — **загрузка**.
- Цена + валюта — опционально (или «по запросу»).
- Слоты: всего / доступно.
- **Дата выхода.**
- **Площадки** (YouTube / Kinodaran / TV / ...).
- **Дедлайн приёма заявок.**
- Порядок сортировки, статус (активен/скрыт).

### 6.4. Портфолио (CRUD)
- Кейсы: название, описание (3 языка).
- **Фото** — загрузка (несколько).
- **Видео** — ссылка YouTube **или** загрузка файла.
- Сортировка.

### 6.5. Партнёры (CRUD)
- Загрузка **логотипа**, ссылка на сайт партнёра, порядок в слайдере.

### 6.6. Заявки
- Список всех заявок с деталями (имя, телефон, email, компания, проект, бюджет, сообщение, дата).
- Статусы: новая / в работе / закрыта.
- Заметки менеджера (опционально).
- Экспорт в CSV/Excel (опционально, желательно).
- **Email-уведомление** при поступлении новой заявки.

### 6.7. Контакты и соцсети
- Редактирование: телефон, WhatsApp, Telegram, email, ссылки на соцсети.

### 6.8. Настройки
- Смена пароля, базовые контактные данные.

## 7. Структура данных (модели БД)

> Все мультиязычные поля хранятся в трёх вариантах (`_ru`, `_en`, `_hy`).

**settings** — `key`, `value` *(хэш пароля админа, контакты, соцсети, язык по умолчанию)*

**content** — `key`, `ru`, `en`, `hy` *(тексты лендинга)*

**projects (films)** — `id`, `title_*`, `genre_*`, `description_*`, `placement_type_*`, `poster`, `price`, `currency` (nullable), `slots_total`, `slots_available`, `release_date`, `platforms[]`, `booking_deadline`, `sort_order`, `is_active`, `created_at`

**portfolio** — `id`, `title_*`, `description_*`, `images[]`, `video_type` (youtube | file), `video_url` / `video_file`, `sort_order`, `created_at`

**partners** — `id`, `name`, `logo`, `url`, `sort_order`

**applications** — `id`, `name`, `phone`, `email`, `company`, `project_id`, `project_title`, `budget`, `message`, `consent`, `status`, `created_at`

## 8. Дизайн и UI/UX

- **Стилистика:** в духе **Netflix** — тёмная тема, кинематографичность, крупная типографика, карточки с постерами и hover-эффектами, плавные анимации.
- **Hero:** анимированный, с **параллакс-эффектом** и сменой кадров при скролле.
- **Брендинг:** создаётся **с нуля** (логотипа/цветов нет). В дизайне предложить тёмную премиальную палитру (чёрный/графит + акцентный цвет, например красный/золотой), кинематографичные шрифты.
- **Компоненты:** генерируются через **21st.dev Magic** → требуется стек, который с ним идеально работает (см. раздел 9).
- **Адаптивность:** mobile-first, безупречно на телефоне, планшете, десктопе.
- **Доступность:** достаточный контраст, alt-тексты, читаемость.

## 9. Технические требования

### 9.1. Стек
Подобран для **максимальной совместимости с 21st.dev Magic** (генерирует компоненты React + Tailwind):

- **Frontend + Backend:** **Next.js** (App Router) — единый проект, server actions / API routes для бэкенда.
- **Стилизация:** **Tailwind CSS** + **shadcn/ui** (идеально сочетается с Magic).
- **Анимация:** Framer Motion (параллакс, плавные переходы).
- **ORM/БД:** Prisma + **MySQL** (нативно поддерживается Hostinger) либо SQLite/PostgreSQL — на усмотрение при разработке.

### 9.2. Хостинг
- **Hostinger.** Учесть: для Next.js нужен тариф с поддержкой **Node.js** (Business / Cloud / VPS). MySQL предоставляется хостингом.
- Для тяжёлых видео рекомендуется **YouTube/внешнее хранилище**, а на сервере хранить изображения и короткие ролики (экономия места и трафика на shared-хостинге).

### 9.3. Загрузка файлов
- Изображения (постеры, логотипы, фото портфолио) — на сервер, с оптимизацией (`next/image`).
- Видео — YouTube-embed (приоритет) или загрузка файлом.

### 9.4. Email
- Уведомления о новых заявках через **SMTP** (почта Hostinger) — Nodemailer.

### 9.5. Безопасность
- HTTPS, хэширование пароля админа (bcrypt), httpOnly-сессии.
- Защита формы от спама (honeypot / captcha), rate limiting на форму и логин.
- Валидация данных на сервере.
- Согласие на обработку ПД, регулярные бэкапы БД.

### 9.6. Прочее (нефункциональные)
- **Производительность:** оптимизация изображений, lazy-load видео.
- **Браузеры:** актуальные Chrome, Safari, Firefox, Edge + мобильные.
- **SEO (рекомендуется):** meta-теги, Open Graph, sitemap, hreflang.
